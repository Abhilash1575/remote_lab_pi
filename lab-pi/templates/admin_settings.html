Serial Ports

Define every MCU this Pi talks to — e.g. one port for a teacher-only readout board and another for the student-interactive board. Ports with "Student visible" off still connect and feed data in the background, but never show up in the student UI. Exactly one port should be the "Primary target" that slider/button commands are written to.

This does not include the Oscilloscope — that's a completely separate system with its own auto-detected port, own worker process, and own trigger/timebase controls on the Oscilloscope view. Students can never connect, disconnect, or repoint it through anything configured here, and a port entry that matches the scope's port is rejected.

The dropdown lists currently-detected /dev/serial/by-id/... paths — stable across reboots/replugs, unlike /dev/ttyUSB0//dev/ttyACM0 which are assigned by detection order. If a board doesn't show up (e.g. two identical boards whose chip has no unique serial, so only one gets a by-id entry), type a path manually — e.g. a /dev/serial/by-path/... one, stable as long as it stays in the same physical USB port.

A port with a fixed path + Auto-connect on (and Student visible off) is the typical setup for a teacher-only readout board.
